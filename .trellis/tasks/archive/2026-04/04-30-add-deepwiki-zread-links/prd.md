# Add DeepWiki and Zread links to summary and final pages

## Goal

Add reader-facing DeepWiki and Zread links for each GitHub project on both the
overview summary page and the generated final project report pages.

## What I Already Know

- The overview page is `site/index.html`, generated by
  `tools/generate_project_index.py` from `site/projects.json`.
- Final project pages under `site/projects/<slug>/index.html` are generated by
  `tools/generate_final_insight_reports.py` and
  `tools/generate_gbrain_insight_report.py`.
- Existing project records already contain GitHub repository URLs as `repo`
  or `url`, so DeepWiki and Zread links can be derived without adding registry
  fields.

## Requirements

- Add DeepWiki and Zread links to every project row on the overview page.
- Add GitHub, DeepWiki, and Zread resource links to every generated final
  project report page.
- Ensure the `project-insight-analysis` skill template/instructions also
  include the same resource-link requirement for newly generated reports.
- Derive external resource links only from valid GitHub repo URLs.
- Preserve existing report/index layout, escaping behavior, and generator-owned
  output workflow.

## Acceptance Criteria

- [x] Overview project rows render `报告`, `Markdown`, `GitHub`, `DeepWiki`, and
  `Zread` links where applicable.
- [x] Final project report pages render `GitHub`, `DeepWiki`, and `Zread`
  links near the hero content.
- [x] The project insight skill template exposes a `{{RESOURCE_LINKS}}`
  insertion point and instructs future report generation to fill it.
- [x] Non-GitHub or malformed URLs do not produce broken DeepWiki/Zread links.
- [x] Generated `site/` artifacts are reproducible from the Python generators.
- [x] Existing verification scripts pass.

## Definition of Done

- Python generators compile.
- Final reports and overview page are regenerated.
- Existing report validator and browser verification are run where available.
- Trellis spec update is considered before wrap-up.

## Technical Approach

- Keep `site/projects.json` schema unchanged and compute links from the existing
  GitHub URL fields.
- Use small generator-local helpers for GitHub repository parsing and external
  resource URL generation.
- Use escaped text/URLs in generated HTML and client-rendered index markup.

## Decision (ADR-lite)

**Context**: The registry contract currently treats `site/projects.json` as a
small static data source. Adding optional link fields would require updating the
registration CLI and registry contract.

**Decision**: Derive DeepWiki and Zread links from existing GitHub URLs instead
of adding persistent registry fields.

**Consequences**: This keeps the feature low-maintenance and applies to future
GitHub projects automatically. Non-GitHub project URLs will simply omit these
derived links unless a future task adds explicit override fields.

## Out of Scope

- Adding new registry fields or project registration CLI flags.
- Validating external pages at runtime in the browser.
- Redesigning the overview or report page layouts.
