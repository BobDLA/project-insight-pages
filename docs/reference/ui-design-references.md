# UI 设计参考与落地要求

本项目的 HTML 报告排版参考了 `ui-ux-pro-max-skill` 的 UI/UX 规则，但最终采用适合“项目洞察报告”的克制技术报告风格，而不是营销落地页风格。

## 外部参考

- GitHub：<https://github.com/nextlevelbuilder/ui-ux-pro-max-skill>
- 仓库定位：面向多平台和多框架生成专业 UI/UX 的 AI skill。
- 本项目使用方式：吸收其设计系统、响应式、可访问性、交付检查等原则，不照搬 landing page、强视觉营销或行业模板。

## 本项目采用的 UI 要求

- **信息优先**：首屏必须说明项目名、定位、采用判断和关键事实，不能做成空泛海报。
- **稳定网格**：页面使用一致的 max-width、section 间距、卡片间距和左边界对齐。
- **可扫描文本**：大段说明拆成 label/value、bullet、表格或流程图；每个条目表达一个判断。
- **清晰层级**：通过字号、字重、间距和对比建立层级；加粗只用于关键判断、差异点和行动。
- **克制配色**：使用语义色 token，避免大面积装饰渐变、强营销色和随机阴影。
- **图表可读**：项目机制图必须是连贯画布，保留结构化图源；手机端提供纵向 fallback。
- **响应式验证**：桌面、1080px 竖向截图、390px 手机视口都不能出现页面级横向滚动。
- **可访问性底线**：正文不低于 16px，行高 1.5-1.75；导航/按钮触控高度不低于 44px；焦点态可见；动效尊重 `prefers-reduced-motion`。

## 不采用的部分

- 不把页面做成产品营销 landing page。
- 不引入独立 infographic HTML。
- 不使用无证据的演示图片替代结构化图源。
- 不使用自动关键词加粗或装饰性卡片堆叠来制造“信息密度”。
